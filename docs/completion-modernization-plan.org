** What is still live

- =KK= is now bound to =consult-imenu= in [[file+emacs:/Users/jay/emacs/emacs-settings/prelude-key-chord.el][📄 /Users/jay/emacs/emacs-settings/prelude-key-chord.el]].

** Why this matters

This kind of mixed state is riskier than an obviously old package that is simply still used.

Possible outcomes of leaving it alone:
- bindings appear to work until package loading changes
- future cleanup gets harder because "excluded" does not mean "unused"
- it becomes unclear whether Helm is intentionally part of the workflow or just surviving by inertia

** Safe conclusion for now

Do not remove Helm-related packages yet.

Instead, treat this as a clarification task for a future checkpoint:
- decide whether =helm-imenu= and =helm-M-x= are still genuinely preferred
- if yes, stop pretending Helm is excluded and document that exception clearly
- if no, replace those bindings with modern alternatives before cleaning up Helm references
