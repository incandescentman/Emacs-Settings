Goal: make table wrapping predictable for proposal workflows while preserving a per-table escape hatch.

* Positioning: Classic vs Tabularray

- The tabularray class family is intentionally opt-in and non-breaking.
- Expected outcome for typical proposal tables is visual parity with existing classes, not a dramatic redesign.
- Main value is structural:
  - cleaner conversion internals,
  - better stability for edge-case tables,
  - easier future maintenance.
- Practical guidance:
  - Use classic classes by default.
  - Use tabularray classes when a real table issue appears (overflow, continuation behavior, fragile layout cases).

* Current Behavior (Post-Refactor)

** Defaults
- For classes =elegant=, =elegant-garamond=, and =beautiful-racket=:
  - Plain Org table export that would normally emit =tabular= is converted to =tabularx=.
  - Columns are mapped to left-aligned wrapping columns.
  - Booktabs-style horizontal rules are injected: =\toprule=, =\midrule=, =\bottomrule=.

** Opt-Out
- Per-table opt-out is supported:
  #+begin_example
  #+ATTR_LATEX: :wrap nil
  | ... |
  #+end_example
- With =:wrap nil=, export remains native =tabular= for that table.

** Shared Architecture
- One shared advice now wraps =org-latex-table=:
  - =jay/latex-wrap-org-tables-advice= in [[file+emacs:/Users/jay/emacs/emacs-settings/latex-templates/jay-latex-table-wrap.el][latex-templates/jay-latex-table-wrap.el]]
- Each class file only registers itself:
  - =(jay/latex-register-wrap-class "elegant")=
  - =(jay/latex-register-wrap-class "elegant-garamond")=
  - =(jay/latex-register-wrap-class "beautiful-racket")=

This replaced three copy-pasted class-specific advice blocks.
